# Token Governor v0.2.0 - Observability, Guardrails, and CI Automation

- Tag: `v0.2.0`
- Release Date: `2026-03`

Token Governor v0.2.0 introduces a production-grade observability and guardrail automation system for LLM runtime cost governance.

This release expands the project from a benchmark comparison tool into a fully automated monitoring and regression-detection platform, including daily benchmarks, guardrails, alerting, trend visualization, and CI automation.

## Added

### Daily Guardrail Benchmark

- Daily lightweight benchmark workflow (limit=20)
- Automatic regression detection for:
- Success rate
- Token usage
- Latency

### Guardrail Failure Automation

- Automatic issue creation when guardrails fail
- Same-day issue deduplication
- Automatic `@maintainers` mention

### Notification Integrations

Guardrail failures now trigger optional notifications via:

- Slack
- DingTalk
- Email

### Benchmark Observability Pipeline

New scripts and workflows for runtime observability:

- `append_benchmark_history.py`
- `generate_trends.py`
- `generate_badges.py`
- `report_kpi.py`

These scripts enable:

- Historical benchmark persistence
- Trend analysis
- README badge generation
- KPI summaries

### Trends Dashboard and GitHub Pages

A new trends visualization pipeline has been added.

Features:

- Historical benchmark charts
- Success rate trend
- Token usage trend
- Latency trend
- Fallback trigger trend

Automatic publishing via GitHub Pages.

Pages entry:

- `docs/index.html`
- `docs/trends/trends.html`

## Improved

5. Trend generation
6. KPI report generation
7. Badge generation

This provides a complete observability pipeline for LLM runtime evaluation.
